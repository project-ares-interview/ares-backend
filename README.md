# ARES Backend Project

## 1. 프로젝트 개요

ARES Backend Project는 AI 기술을 활용하여 사용자의 경력 개발 및 채용 준비 과정을 지원하는 백엔드 시스템입니다. 이 프로젝트는 이력서 분석, 면접 시뮬레이션, 맞춤형 피드백 제공 등을 통해 사용자가 직무 역량을 강화하고 성공적인 취업을 할 수 있도록 돕는 것을 목표로 합니다.

## 2. 주요 기능

### 2.1. 이력서 피드백 (Resume Feedback)

사용자의 이력서와 지원 직무 정보를 분석하여 이력서의 완성도를 높이고 직무 적합성을 강화할 수 있도록 돕는 기능입니다.

*   **입력**:
    *   JD (Job Description, 직무기술서)
    *   Resume (이력서)
    *   Research (기업 사업보고서, 산업 동향 보고서 등)
*   **분석 및 피드백**:
    *   입력된 정보를 바탕으로 이력서의 강점과 약점을 파악합니다.
    *   이력서 내용, 구성, 표현 방식 등에 대한 상세 피드백을 제공합니다.
    *   NCS (국가직무능력표준)와 연계하여 직무 역량 관점에서 이력서 개선 가이드를 제시합니다.
*   **현재 RAG 적용 현황**:
    *   JD, 이력서, 리서치 자료는 현재 LLM에 컨텍스트 정보로 전달되어 분석에 활용됩니다. 즉, LLM이 이 모든 텍스트를 직접 읽고 이해하여 피드백을 생성하는 방식입니다.
    *   NCS 관련 자료는 별도의 검색 기능을 통해 RAG와 유사한 방식으로 활용되어 피드백에 포함됩니다.
    *   **참고**: JD와 이력서 내용은 현재 RAG의 '검색' 단계(즉, 특정 질문에 가장 관련성 높은 부분을 찾아 LLM에 제공하는 방식)로 활용되고 있지는 않습니다. 전체 문서가 LLM에 전달되는 경향이 있습니다.

### 2.2. 면접 피드백 (Interview Feedback)

사용자가 실제 면접 상황을 시뮬레이션하고, 답변에 대한 즉각적인 피드백을 받아 면접 역량을 강화할 수 있도록 돕는 기능입니다.

*   **사용자 플로우**:
    1.  **면접 시작**: 사용자가 지원할 회사명, 직무를 입력하고, 면접 난이도(쉬움, 보통, 어려움)를 선택합니다. 시스템은 이 정보를 바탕으로 첫 면접 질문을 생성하여 사용자에게 제시합니다.
    2.  **답변 및 피드백**: 사용자가 질문에 답변하면, 시스템은 답변 내용을 분석하여 점수, 상세 피드백, 그리고 모범 답변을 제공합니다. 이와 함께 다음 면접 질문(꼬리 질문)을 생성하여 대화를 이어갑니다.
    3.  **면접 종료 및 리포트**: 면접 세션이 종료되면, 전체 면접 과정에 대한 종합적인 분석이 담긴 최종 리포트를 사용자에게 제공합니다.
*   **현재 RAG 적용 현황**:
    *   **NCS RAG**: 면접 질문 생성, 답변 분석, 피드백 제공 시 NCS 관련 컨텍스트가 활용되어 직무 역량 기반의 평가가 이루어집니다.
    *   **회사 사업보고서 RAG**: 현재 **'어려움(hard)' 난이도 모드에서만** 회사 사업보고서 RAG가 활성화되어 있습니다. 이 모드에서는 사업보고서에서 검색된 정보를 바탕으로 질문을 생성하고, 답변을 분석하며, 꼬리 질문을 만들고, 최종 리포트를 작성하는 데 활용됩니다.
    *   **참고**: JD, 이력서, 리서치 자료는 면접 세션 시작 시 컨텍스트 정보로 전달되지만, 회사 사업보고서 RAG처럼 세분화된 검색 방식으로 면접의 모든 단계에 직접적으로 활용되지는 않습니다.
*   **대체 피드백 구조 (MCP 관련 기능)**: STARC 방식 외에, 별도의 피드백 구조가 적용되어 있습니다. 이는 면접 답변 분석 및 피드백 제공 시 다양한 관점을 제공합니다.

## 3. 핵심 기술 및 아키텍처

*   **LLM (Large Language Model)**: Azure OpenAI 서비스를 활용하여 질문 생성, 답변 분석, 피드백 제공 등 핵심 AI 기능을 수행합니다.
*   **RAG (Retrieval-Augmented Generation)**: 특정 컨텍스트(NCS, 회사 사업보고서 등)를 LLM에 제공하여 답변의 정확성과 관련성을 높이는 데 사용됩니다.
*   **Django / Django REST Framework**: 백엔드 API 개발 및 데이터 관리를 위한 프레임워크로 사용됩니다.
*   **Azure Blob Storage**: 대량의 문서(사업보고서, 이력서 등) 저장을 위한 클라우드 스토리지 서비스입니다.
*   **Azure AI Search (예상)**: RAG를 위한 문서 인덱싱 및 검색에 활용될 수 있습니다. (현재는 회사별 인덱스 방식)

## 5. 향후 개선 사항 및 논의점

프로젝트의 품질과 효율성을 높이기 위해 다음과 같은 개선 사항과 논의점을 제안합니다.

### 5.1. RAG 시스템의 전면적인 통합 및 확장

*   **이력서/JD에 대한 RAG 적용**:
    *   **개선 방향**: 이력서와 JD 내용을 인덱싱하여, 특정 피드백 항목에 대해 해당 문서에서 가장 관련성 높은 부분을 검색하여 LLM에 제공하는 RAG 방식을 도입합니다.
    *   **기대 효과**: LLM이 핵심 정보에 집중하여 더 정확하고 구체적인 이력서 피드백을 생성할 수 있으며, LLM의 컨텍스트 창 한계를 극복합니다.
*   **리서치 자료(사업보고서 등)의 RAG 통합**:
    *   **개선 방향**: 사용자가 업로드한 리서치 자료를 RAG 시스템에 통합하여 인덱싱하고, 이력서 분석 시 기업의 비전, 인재상 등과 관련된 컨텍스트를 검색하여 활용합니다.
    *   **기대 효과**: 이력서 피드백이 지원 기업의 특성과 문화에 더욱 맞춤화된 형태로 제공됩니다.
*   **면접 피드백의 전면적인 RAG 통합**:
    *   **'쉬움(easy)' 모드에 회사 사업보고서 RAG 활성화**: 현재 'hard' 모드에만 적용되는 회사 사업보고서 RAG를 '쉬움' 모드에도 적용하여, 난이도와 관계없이 회사 맞춤형 면접 경험을 제공합니다. (코드 변경: `use_rag_mode = (difficulty == 'hard' or difficulty == 'easy')`)
    *   **JD, 이력서, 리서치(사업보고서 포함)의 전면적인 RAG 활용**: 면접의 모든 단계(질문 생성, 답변 분석, 피드백, 모범 답변, 꼬리 질문, 최종 리포트)에서 JD, 이력서, 리서치를 RAG 컨텍스트로 활용하여 면접의 개인화 및 정확도를 극대화합니다.

### 5.2. RAG 시스템 아키텍처 최적화 및 비용 관리

*   **현재 아키텍처**: 회사별로 별도의 검색 인덱스를 생성하는 방식.
*   **제안하는 최적화 방안**:
    *   **단일 컨테이너 사용**: 모든 기업의 사업보고서 및 기타 리서치 자료를 하나의 Blob Storage 컨테이너에 통합 저장.
    *   **단일 통합 검색 인덱스 구축**: 모든 기업의 사업보고서 데이터를 포함하는 하나의 통합된 검색 인덱스를 구축하고, `company_name`과 같은 메타데이터 필터링을 통해 특정 기업 정보 검색.
    *   **Upsert 방식 도입**: 새로운 데이터 추가/업데이트 시 `upsert` 기능을 활용하여 변경된 부분만 효율적으로 인덱스에 반영.
*   **비용 고려사항**:
    *   RAG 시스템의 전면적인 통합은 데이터 저장, 인덱싱, 검색, LLM 추론 등 여러 단계에서 비용 증가를 야기할 수 있습니다.
    *   **비용 완화 전략**: 계층형 RAG, 최적화된 인덱싱, 효율적인 검색, LLM 사용 최적화, 클라우드 자원 최적화 등을 통해 비용을 효과적으로 관리할 수 있습니다.
    *   **가치 vs. 비용**: RAG를 통한 서비스 품질 향상(개인화, 정확성, 효율성)이 가져올 장기적인 가치와 비용 증가분을 비교하여 최적의 균형점을 찾아야 합니다.

### 5.3. 기타 논의점

*   **기술 로드맵**: RAG 아키텍처 개선 및 기능별 RAG 적용 확대를 위한 구체적인 기술 로드맵과 개발 일정을 수립.
*   **데이터 관리 및 운영**: RAG 시스템의 효율적인 운영을 위한 데이터 수집, 전처리, 인덱싱, 모니터링 등 전반적인 데이터 관리 전략 수립.
*   **사용자 경험 (UX) 향상**: RAG 통합을 통해 사용자가 어떤 부분에서 가장 큰 가치를 느끼고, 어떤 방식으로 새로운 기능을 제공할지 UX/UI 관점에서도 함께 고민.
